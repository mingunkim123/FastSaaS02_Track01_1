## CRITICAL 수정사항

### 1. 세션 소유권 검증 누락 수정

**문제**

`POST /api/ai/action` 엔드포인트에서 요청에 포함된 `sessionId`가 실제로 현재 인증된 사용자 소유인지 확인하지 않았다. 이로 인해 공격자가 다른 사용자의 `sessionId`를 알고 있을 경우 해당 세션에 메시지를 주입할 수 있는 취약점이 존재했다.

**수정**

`getSession(db, sessionId, userId)` 호출을 추가하여 세션 조회 시 소유권을 동시에 검증하도록 변경했다. 세션이 존재하지 않거나 소유자가 일치하지 않을 경우 HTTP 403을 반환한다.

**관련 파일**

- `backend/src/routes/ai.ts`
- `backend/src/services/sessions.ts`

### 2. 하드코딩된 Supabase URL 제거

**문제**

`https://uqvnepemplsdkkawbmdc.supabase.co` 주소가 소스코드에 직접 하드코딩되어 있었다. 이 값이 버전 관리 시스템에 노출되면 프로젝트 식별이 가능해지고, 환경별 설정 변경이 불가능하다.

**수정**

Supabase URL을 `c.env.SUPABASE_URL` 환경변수를 통해 주입받도록 변경했다. `Env` 타입 정의에 `SUPABASE_URL: string` 필드를 추가했으며, `wrangler.jsonc`에 플레이스홀더를 추가하여 배포 전 실제 값으로 교체하도록 안내한다.

**관련 파일**

- `backend/src/middleware/auth.ts`
- `backend/src/db/index.ts`
- `backend/wrangler.jsonc`

## MEDIUM 수정사항

### 5. AI 엔드포인트 Rate Limiting 추가

**문제**

AI 처리 엔드포인트에 요청 횟수 제한이 없었다. 악의적이거나 오작동하는 클라이언트가 짧은 시간에 대량의 요청을 보낼 경우 LLM API 비용이 급격히 증가할 수 있었다.

**수정**

신규 미들웨어 파일 `backend/src/middleware/rateLimit.ts`를 생성하고, 다음 제한을 적용했다.

| 엔드포인트 | 제한 |
|-----------|------|
| `POST /api/ai/action` | 사용자당 분당 20회 |
| `POST /api/sessions/:sessionId/messages` | 사용자당 분당 20회 |
| `POST /api/reports` | 사용자당 분당 10회 |

제한 초과 시 HTTP 429를 반환한다.

**주의사항**: 현재 구현은 인메모리 방식이다. Cloudflare Workers는 멀티 isolate 환경에서 동작하므로 isolate 간 상태가 공유되지 않는다. 엄격한 전역 제한이 필요한 경우 Cloudflare Durable Objects를 사용하는 방식으로 전환을 권장한다.

**관련 파일**

- `backend/src/middleware/rateLimit.ts` (신규)
- `backend/src/routes/ai.ts`
- `backend/src/routes/sessions.ts`
- `backend/src/routes/reports.ts`

### 6. 에러 핸들링 개선

**문제**

라우트 핸들러의 `catch` 블록이 모든 예외를 동일하게 처리하여 HTTP 500을 반환하고 있었다. 클라이언트 입력 오류와 서버 내부 오류를 구분하지 않았고, 에러 메시지에 내부 스택 트레이스나 시스템 정보가 포함될 가능성이 있었다.

**수정**

에러 타입에 따라 응답 코드를 분류하도록 개선했다.

| 에러 유형 | HTTP 응답 |
|----------|----------|
| ZodError, SyntaxError (입력 검증 실패) | 400 Bad Request |
| LLM 호출 실패, 네트워크 타임아웃 | 503 Service Unavailable |
| DB 오류, 기타 내부 오류 | 500 Internal Server Error |

클라이언트에 반환하는 에러 메시지에는 내부 구현 세부사항을 포함하지 않는다.

**관련 파일**

- `backend/src/routes/ai.ts`
- `backend/src/routes/sessions.ts`

## LOW 수정사항

### 8. CORS 환경변수 기반 설정

**문제**

CORS 허용 origin이 `http://localhost:3000`으로 소스코드에 하드코딩되어 있었다. 배포 환경에 따라 다른 origin을 허용해야 할 때 코드 수정이 필요했다.

**수정**

`ALLOWED_ORIGINS` 환경변수에서 쉼표로 구분된 origin 목록을 읽어 동적으로 적용하도록 변경했다. 해당 환경변수가 설정되지 않은 경우 기존 하드코딩 값으로 폴백한다.

**관련 파일**

- `backend/src/index.ts`
- `backend/wrangler.jsonc`
- `backend/src/db/index.ts`

### 9. parseInt NaN 처리

**문제**

`parseInt()` 호출 결과가 `NaN`인 경우 그대로 DB 쿼리에 전달될 수 있었다. 이는 예상치 못한 쿼리 동작이나 런타임 오류를 유발할 수 있었다.

**수정**

`parseInt(str) || defaultValue` 패턴을 적용하여 파싱 실패 시 안전한 기본값으로 폴백하도록 했다. 페이지 크기, 오프셋 등의 값에는 최솟값/최댓값 클램핑도 추가했다.

**관련 파일**

- `backend/src/routes/reports.ts`
- `backend/src/routes/ai.ts`

### 10. 민감 정보 console.log 제거

**문제**

JWT `kid`, JWKS URL, `userId` 등의 민감한 값이 `console.log`를 통해 로그에 출력되고 있었다. 이는 Cloudflare Workers 로그 조회 권한이 있는 운영자에게 불필요한 정보를 노출시킨다.

**수정**

디버그 목적의 `console.log` 호출을 제거했다. 오류 추적에 필요한 경우 `console.error`만 유지하되, 출력 내용에 JWT 토큰, userId, URL 등 민감 정보를 포함하지 않도록 정리했다.

**관련 파일**

- `backend/src/middleware/auth.ts`
- `backend/src/services/clarifications.ts`

# 신규 환경변수 등록

# 타입 체크

npm run deploy
